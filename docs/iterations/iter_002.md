# Iteration 002 - Day 2

## 任务概述

**任务名称**：数据标注实践与审核流程  
**所属周**：Week 1 - 数据标注与数据分析基础  
**目标**：掌握数据标注操作流程，熟悉标注工具和审核流程，完成 50 条文本标注，并记录易错点。

## 背景/目的

- 熟悉实际标注操作和工具使用
- 理解标注审核流程和常见错误类型
- 提高标注准确率，为后续数据分析和模型训练准备高质量数据

## 要解决的问题

- 学会高效进行大规模标注
- 掌握边界判断、标签一致性和异常情况处理
- 了解标注复核流程，发现和纠正易错标签

## 参考资料

- 开源数据集：SQuAD、CoQA
- 自行模拟或采集的 50 条文本样本
- 标注工具文档（如 Label Studio、doccano）
- Day 1 标注规则文档：`day1_annotation_practice.md`

## 任务要求

- 使用标注工具完成 50 条文本标注
- 按 Day 1 规则执行标注，尤其注意边界判断
- 标注后进行自检或互检，记录易错标签和纠正方法
- 生成标注日志文档，包含每条样本的标注结果和发现的问题

## 实现步骤

1. 准备 50 条文本样本（可选 SQuAD/CoQA 或模拟数据）
2. 使用标注工具完成标注，保证任务类型覆盖：分类、抽取、BIO
3. 标注完成后进行校对或复核，记录发现的错误或边界问题
4. 将标注日志整理成 Markdown 文档：
   - 每条样本包含原始文本、标注结果、发现的问题及修改理由
   - 总结易错标签和常见边界问题

## 验收标准

- 标注数量：50 条完整标注
- 标注准确率 ≥ 90%
- 标注日志中清楚记录边界问题和纠正方法
- 标注结果符合 Day 1 的规则文档要求
