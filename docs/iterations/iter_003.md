# Day 3 - 数据一致性检查与标注质量分析

## 任务目标与执行说明

- **任务名称**：数据一致性检查与标注质量分析
- **所属阶段**：Week 1 / Day 3
- **目标**：学会使用 Python 对标注数据进行统计分析，包括重复值、缺失值和标注一致性检查，为数据质量评估提供基础。
- **参考文档**：`day2_annotation_practice.md`，`docs/iterations/iter_002.md`
- **任务范围**：对前两天标注生成的 CSV/Excel 数据进行完整分析，输出统计报告。

---

## 一、任务步骤

1. **数据导入**
   - 使用 Pandas 导入 Day 2 标注生成的 CSV 或 Excel 文件。
   - 核验列名、数据类型是否符合预期（例如：`ID`、`任务类型`、`原始文本`、`标注结果`、`发现的问题及纠正方法`）。

2. **重复值与缺失值统计**
   - 检查是否存在重复 ID 或重复文本。
   - 统计每列缺失值数量及比例。

3. **标注一致性分析**
   - 对分类任务：
     - 统计不同标签 (`POS` / `NEG` / `NEU`) 的分布。
   - 对实体抽取：
     - 检查相同文本出现不同实体标注的情况。
   - 对 BIO 序列：
     - 验证每条序列是否符合合法 BIO 规范（B/I/O 开头规则、连续性）。
   - 可选：计算简单一致性指标（如重复标注的一致率）。

4. **报告生成**
   - 输出表格，展示：
     - 总样本数
     - 每列缺失值数量
     - 重复样本及重复率
     - 标签分布统计
     - 一致性异常样本列表
   - 可使用 CSV 或 Excel 保存分析结果，便于后续复核。

---

## 二、任务要求

- 编写 Python 脚本（Jupyter Notebook 或 `.py` 文件皆可）。
- 脚本可重复执行，对不同批次数据通用。
- 输出分析结果清晰，包含图表或表格。
- 对发现的问题需标注备注（如重复、缺失、不一致样本）。

---

## 三、验收标准

1. 脚本能顺利导入 CSV/Excel 数据并完成分析。
2. 统计报告覆盖：
   - 样本总数
   - 缺失值和重复值数量
   - 分类、实体和 BIO 的分布统计
   - 一致性问题样本
3. 输出格式清晰、可直接用于数据质量评估。
4. 分析结果与 Day 2 标注内容一致，便于识别异常或易错标签。
