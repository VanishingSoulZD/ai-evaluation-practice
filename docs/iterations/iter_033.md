# Iteration 033 — 配置 Label Studio，跑通第一个项目

## 1. 本次迭代目标

本轮的目标是完整跑通 **Label Studio 的最小可用闭环**：

1. 安装并启动 Label Studio；
2. 创建一个文本标注项目；
3. 导入第 31 天整理好的样本；
4. 根据第 32 天的标注规范配置标注界面；
5. 完成第一轮人工标注；
6. 导出结果，并确认后续脚本可以正常使用。

本轮的重点不是“把工具点开”，而是要理解它在整个评测链路中的位置：

> **样本 → 标注项目 → 人工标注 → 导出结果 → 后续分析与评测**

---

## 2. 本轮为什么先做 Label Studio

Label Studio 是本项目的第一个完整标注平台实践点。先做它的原因是：

- 它可以把原始样本直接变成可标注项目；
- 它支持文本相关任务的自定义界面；
- 它的导出结果便于进入后续标注分析、统计和评测脚本；
- 它适合作为后续 doccano、Kappa 分析和统一标注流程的参照。

本轮的目标是先掌握一个平台的完整闭环，再把经验迁移到其他工具。

## 5. 本轮执行步骤

### Step 1：安装并启动 Label Studio

先确认本地环境可用，并能正常访问 Label Studio 服务页面。

这一阶段的目标不是部署复杂环境，而是确保：

- 能启动服务；
- 能登录界面；
- 能创建项目；
- 能保存配置。

### Step 2：创建文本标注项目

在 Label Studio 中新建一个文本相关项目。

创建项目时要明确：

- 项目名称；
- 任务类型；
- 使用的输入字段；
- 是否需要参考答案字段；
- 后续导出时需要保留哪些元信息。

### Step 3：导入第 31 天的样本

将第 31 天的样本导入 Label Studio。

导入时要检查：

- JSON / CSV 是否被正确识别；
- 样本字段是否映射正确；
- 是否有缺失字段；
- 是否出现乱码或格式错误。

### Step 4：根据第 32 天的标注规范配置界面

根据已经写好的标注规范，配置项目的标注界面。

配置重点包括：

- 标签按钮；
- 输入框；
- 说明文字；
- 任务模块对应的标注选项；
- 是否需要显示上下文、问题和参考信息。

### Step 5：做第一轮人工标注

按照标注规范，对导入样本完成第一轮人工标注。

标注时要检查：

- 界面是否足够清晰；
- 标签是否容易点错；
- 说明文字是否足够明确；
- 是否存在需要调整的歧义样本。

### Step 6：导出结果并验证格式

完成标注后导出数据，并检查导出的结果是否能供后续脚本使用。

验证重点：

- 导出字段是否完整；
- 标注内容是否可解析；
- 格式是否能与后续分析脚本对接；
- 是否保留了样本 id 和任务信息。

## 9. 本轮执行结论

### 9.1 结论一：Label Studio 是本项目标注链路的重要起点

本轮已经验证，Label Studio 可以作为文本标注项目的标准入口，用于后续的人工作业和数据质检。

### 9.2 结论二：界面配置必须跟规范同步

标注工具不是单独存在的。标注规范决定了标签如何设计，工具界面只是把规范落实到操作层。

### 9.3 结论三：导出格式必须可复用

本轮不仅要完成标注，还要确认导出结构能被后续脚本读取，否则工具闭环不成立。
