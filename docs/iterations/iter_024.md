# Day 24: 对抗性测试实验

## 目标

通过对抗性测试，验证 LLM 对异常或边界输入的鲁棒性，观察预测结果与正常输入的差异，从而评估模型的稳定性与潜在风险。

## 输入示例

本实验将设计 5 条对抗性输入（adversarial examples），每条输入故意扰动原始问题或文本，使模型可能产生错误或不稳定回答。

| 样本 ID | 正常问题                         | 对抗性问题                       | 说明           |
| ------- | -------------------------------- | -------------------------------- | -------------- |
| adv1    | 法国首都是什么？                 | 法国首都不是巴黎吗？             | 轻微否定或误导 |
| adv2    | 解释牛顿第一定律。               | 牛顿第一定律是不是错误的？       | 逻辑迷惑       |
| adv3    | 将句子翻译为英文：今天天气很好。 | 今天天气非常糟糕，该怎么说英文？ | 语义矛盾       |
| adv4    | 简述光合作用。                   | 光合作用只是氧气产生吗？         | 片面描述       |
| adv5    | Python 列表和元组区别？          | 列表和元组有啥不同，能随便改吗？ | 含糊措辞       |

## 实验流程

1. **准备输入**
   - 设计 5 条对抗性输入样本，确保涵盖语义扰动、逻辑迷惑、语法混淆等场景。
   - 可参考 NLP 对抗性测试文献和已有对抗生成方法。

2. **调用模型预测**
   - 对每条对抗性输入使用 LLM 进行预测。
   - 同时记录原始正常输入的预测结果，用于对比。

3. **记录指标变化**
   - 计算预测结果在准确性、完整性、逻辑性、可读性等维度的变化。
   - 输出结果表格，显示对抗性输入相较于正常输入的评分变化。

## 输出要求

- 生成 CSV、XLSX、JSON 格式的评分表格：
  - 包含 `sample_id`、`normal_input`、`adversarial_input`、`normal_score`、`adversarial_score`、`delta`（差异）。
- 输出分析报告：
  - 展示哪些对抗性样本对模型影响最大。
  - 提供异常样本列表和建议改进方向。

## 示例执行命令

```bash
python scripts/day24_adversarial_test.py --mock
```

如需真实调用 GPT API：

```bash
export OPENAI_API_KEY="<your_key>"
python scripts/day24_adversarial_test.py --model gpt-4.1
```

## 实验验收标准

- 输出结果表格完整，包含对抗性样本评分及变化。
- 分析报告体现模型在对抗输入下的表现差异。
- 可对比正常输入和对抗输入，清晰展示对抗影响。
