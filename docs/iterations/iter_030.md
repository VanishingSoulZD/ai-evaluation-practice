# Iteration 030 — 理解 SQuAD、CoQA、GLUE 的样本结构

## 1. 本次迭代目标

本轮的目标是把主流数据集的**输入输出结构**、**标签形式**和**适用能力**理清楚，并据此完成本项目后续样本抽取的统一口径。

本轮要解决的核心问题只有三个：

1. 数据长什么样；
2. 标签长什么样；
3. 适合评测什么能力。

本轮不做模型训练，不做指标计算，不做标注工具配置，重点是为后续标注和评测打地基。

---

## 2. 本轮采用的数据集范围

本轮只看三类主流数据集：

- **SQuAD**：单轮阅读理解 / span answer；
- **CoQA**：多轮上下文相关问答；
- **GLUE**：通用语言理解任务集合。

选择这三类数据集的原因是：

- 它们分别代表了问答、对话问答、通用语言理解三个典型方向；
- 后续可以自然过渡到标注、一致性、自动指标和 judge 评测；
- 它们的样本结构清晰，适合做统一格式设计。

---

## 3. 外部资源入口

本轮只需要浏览官方页面中与样本结构相关的部分，不需要通读全部内容。

### 3.1 SQuAD

- 官方入口：[https://rajpurkar.github.io/SQuAD-explorer/](https://rajpurkar.github.io/SQuAD-explorer/)

需要重点看的内容：

- 问题（question）如何与上下文（context）对应；
- 答案是否是原文 span；
- 一个样本的字段结构；
- 阅读理解任务如何定义“命中答案”。

### 3.2 CoQA

- 官方入口：[https://stanfordnlp.github.io/coqa/](https://stanfordnlp.github.io/coqa/)

需要重点看的内容：

- 对话历史如何影响后续问题；
- 一个轮次中有哪些字段；
- 答案是否依赖前文；
- 多轮上下文问答和单轮阅读理解的区别。

### 3.3 GLUE

- 官方入口：[https://gluebenchmark.com/](https://gluebenchmark.com/)

需要重点看的内容：

- GLUE 是任务集合而不是单一数据集；
- 不同子任务的输入输出形式；
- 分类、相似性判断、蕴含判断等任务的差异；
- 为什么 GLUE 适合评测通用语言理解能力。

## 4. 本轮样本组织原则

本轮要从主流数据集中提取少量样本到本地，并建立统一目录。后续所有迭代都尽量复用这套结构。

### 4.1 建议目录结构

```text
data/day30_raw/
  squad/
  coqa/
  glue/
```

### 4.2 统一组织原则

1. 每个数据集单独一个目录，不混放；
2. 原始数据和整理后数据分开存；
3. 保留来源说明，避免后续样本混淆；
4. 样本量不需要大，但结构必须规范；
5. 后续标注、分析、评测都基于统一字段。

### 4.3 建议的统一字段

为方便后续处理，建议每条样本至少保留以下字段：

- `sample_id`
- `source_dataset`
- `task_type`
- `input`
- `context`（如适用）
- `question`（如适用）
- `reference_answer`（如适用）
- `label`（如适用）
- `notes`

## 5. 本轮执行步骤

### Step 1：浏览 SQuAD 官方页面

先确认以下信息：

- SQuAD 的样本是不是“上下文 + 问题 + 答案”结构；
- 答案是不是原文中的 span；
- 这种结构为什么适合做阅读理解评测。

### Step 2：浏览 CoQA 官方页面

先确认以下信息：

- CoQA 是否包含多轮问题；
- 前一轮回答如何影响后一轮问题；
- 为什么它比 SQuAD 更接近真实对话问答场景。

### Step 3：浏览 GLUE 官方页面

先确认以下信息：

- GLUE 不是单一数据集，而是多个任务集合；
- 不同子任务的标签形式有哪些差异；
- 为什么它适合练“通用语言理解”而不是单一问答能力。

### Step 4：抽取少量样本到本地

从三个数据集里各取少量样本，形成本地的样本集合。

建议做法：

- 每个数据集先保留 5~20 条样本；
- 先关注结构正确，而不是数量多；
- 保留原始字段和整理后字段两份。

### Step 5：写一份任务形式对比文档

把 SQuAD、CoQA、GLUE 的任务形式写成一份对比说明，至少包括：

- 任务类型；
- 输入结构；
- 输出结构；
- 评测关注点；
- 后续适合进入哪一类评测流程。

## 7. 本轮得到的结论

### 7.1 SQuAD 更适合练“答案是否命中证据 span”

SQuAD 的核心特点是上下文明确、答案通常来自原文片段，因此特别适合训练和评估：

- 是否能从证据中定位答案；
- 是否能理解问题和上下文之间的对应关系；
- 是否能在阅读理解任务中准确抽取答案。

### 7.2 CoQA 更适合练“上下文依赖与多轮对话”

CoQA 的核心价值在于它不是孤立单轮问答，而是多轮对话式问答，因此更适合评估：

- 对前文历史的利用能力；
- 多轮上下文记忆；
- 回答是否能结合对话语境进行推理。

### 7.3 GLUE 更适合练“通用语言理解的分类/匹配判断”

GLUE 的价值在于它覆盖多个语言理解任务，不是单一问答数据集，因此更适合评估：

- 句子对关系判断；
- 文本分类；
- 语义相似度判断；
- 更广义的语言理解能力。

## 8. 本轮执行结论

本轮已经确认：

1. 本项目后续样本来源应优先使用主流公开数据集，而不是自行生成样本；
2. 后续统一样本格式时，需要兼容问答、对话问答和分类/匹配类任务；
3. 评测链路的第一步不是写指标，而是先把任务形式和样本结构定义清楚。
