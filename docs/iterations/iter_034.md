原因是：

- doccano 更轻量；
- 更偏传统 NLP；
- 更适合文本分类、NER、句子对关系等标准 NLP 标注；
- 社区中大量公开 NLP 标注项目都基于它；
- 它的数据结构更容易进入传统训练和评测脚本。

因此本轮重点是：

- 理解 doccano 擅长什么；
- 理解它和 Label Studio 的取舍；
- 学会把两者都接入统一评测链路。

# 5. doccano 在评测链路中的位置

本轮需要明确：

```text

    ↓
统一样本格式
    ↓
doccano 标注项目
    ↓
人工标注
    ↓
导出 JSONL / CSV
    ↓
```

doccano 本质上是：

它不是评测工具本身，而是：

# 8. doccano 与 Label Studio 的差异

这是本轮最重要的部分之一。

---

## 8.1 doccano 更适合什么

doccano 更适合：

- 传统 NLP 标注；
- 文本分类；
- NER；
- 句子对关系；
- 结构相对固定的文本任务。

它的特点是：

- 界面轻量；
- 学习成本低；
- 配置简单；
- 导出结构清晰；
- 更偏 NLP 社区传统工作流。

---

## 8.2 Label Studio 更适合什么

Label Studio 更适合：

- 高度可定制任务；
- 多模态任务；
- 复杂字段输入；
- 多组件组合界面；
- 更复杂的人机协作流程。

它的特点是：

- UI 灵活；
- 配置能力强；
- 支持范围广；
- 更适合复杂 AI 数据工程。

---

## 8.3 两者的核心区别

| 对比项     | doccano       | Label Studio      |
| ---------- | ------------- | ----------------- |
| 定位       | 传统 NLP 标注 | 通用 AI 标注平台  |
| 配置复杂度 | 低            | 高                |
| UI 灵活性  | 中            | 很高              |
| 上手速度   | 快            | 中                |
| 适合任务   | NLP 经典任务  | 多模态 + 定制任务 |
| 数据结构   | 更固定        | 更灵活            |
| 工程扩展性 | 中            | 高                |

# 12. 本轮执行结论

## 12.1 结论一：doccano 更偏传统 NLP

它非常适合：

- 分类；
- NER；
- 文本关系；
- 结构固定任务。

但不如 Label Studio 灵活。

---

## 12.2 结论二：Label Studio 更偏 AI 数据工程平台

它更适合：

- 多模态；
- 多字段复杂输入；
- 可定制 UI；
- 更复杂流程。

---

## 12.3 结论三：两者都只是“标注入口”

真正重要的是：

- 统一样本格式；
- 统一标注规范；
- 统一导出结构；
- 能进入后续评测链路。
