# Iteration 026 - Day 26

## 基本信息

- **Day**: 26
- **主题**: 综合流程练习
- **目标**: 练习从标注到指标计算再到报告生成的完整流程，形成可复用流程
- **覆盖范围**: Week1~Week4 学习内容
- **产出**: 可复用的完整流程报告

---

## 背景

经过前三周的任务积累，我们已经掌握了：

1. 文本/对话标注方法与技巧
2. 指标计算和分析方法（accuracy, completeness, logic, readability 等）
3. 报告生成和结果可视化方法

Day 26 的目标是将以上能力整合，模拟完整的从**标注 → 指标计算 → 报告生成**的工作流程，练习在小数据集上完整执行流程，并确保流程可复用。

---

## 任务说明

### 任务概览

1. **标注小数据集**
   - 使用已学标注规范（Week1~Week3）对 5~10 条样本进行手工标注。
   - 样本类型涵盖事实问答、步骤说明、总结型任务、情感支持、技术解释。
   - 输出标注文件格式为 JSON 或 CSV，包含 `sample_id, input, reference_output, scene_note` 等字段。

2. **批量计算指标**
   - 使用之前练习的指标计算方法（Week2~Week3）。
   - 计算每条样本的 accuracy, completeness, logic, readability, overall_score。
   - 对每个维度计算均值和方差，并标注出异常值。
   - 可使用 mock 方法或调用 LLM 自动评分。

3. **生成报告**
   - 生成 JSON + CSV + XLSX 版本的综合分析报告。
   - 报告包含：
     - 总样本数、成功/失败数量
     - 各维度均值与方差
     - 异常样本列表
     - 最大影响样本及其 delta_overall_score
   - 确保报告格式可复用到后续迭代和其他数据集。

---

### 任务要求

- 小数据集标注要覆盖不同场景和任务类型
- 指标计算必须保证可复现，并可对新数据快速批量执行
- 报告生成需可视化关键统计信息，结构清晰、可直接存档
- 所有文件命名规范统一：
  - `day26_annotation.json`
  - `day26_metrics.json`
  - `day26_metrics.csv`
  - `day26_metrics.xlsx`
  - `day26_analysis_report.json`

---

### 具体实现步骤

1. **准备样本**
   - 从 Week1~Week4 已学示例中抽取 5~10 条多场景样本
   - 确认样本类型覆盖全面
2. **标注数据**
   - 使用 JSON/CSV 标注工具或脚本生成标注文件
3. **指标计算**
   - 使用已实现的 `mock_score` 或 `call_llm_score` 方法计算每条样本指标
   - 生成含 delta 值的 CSV/JSON 行记录
4. **报告生成**
   - 调用 `analyze(rows)` 方法生成综合报告 JSON
   - 输出可复用 XLSX
   - 确认报告结构完整，可用于后续迭代或团队复用
5. **检查和验证**
   - 核对每条样本的标注和计算结果
   - 确认整体指标合理，异常值标注正确
   - 确认报告文件可被其他脚本直接读取

---

### 验收标准

- **产出文件完整**：标注文件 + 指标文件 + 分析报告（JSON/CSV/XLSX）
- **指标计算准确**：均值和方差正确计算，异常值正确识别
- **流程可复用**：脚本和方法可直接用于其他小数据集
- **报告清晰**：包含总览、维度统计、异常样本和最大影响样本
