# Iteration 025: 仿真环境评估

## 任务概述

**目标**：搭建一个仿真环境，对 LLM 的对话能力进行模拟用户交互测试，并记录对话质量指标。  
通过简单模拟对话，评估模型在特定场景下的响应准确性、完整性、逻辑性和可读性。

**背景**：  
在实际应用中，LLM 对话系统面临多种用户输入，尤其是复杂或模糊的交互场景。通过仿真环境，可以在低成本条件下快速评估模型性能，并为后续优化提供量化依据。

---

## 知识点参考

1. **模拟用户交互方法**：如 rule-based 用户模拟、模板输入、对话脚本等。
2. **对话生成与评测文献**：
   - 对话生成模型评测指标（BLEU、ROUGE、METEOR、BERTScore 等）
   - 人机对话仿真环境研究
3. **指标设计**：
   - `accuracy`：回答是否正确
   - `completeness`：回答信息是否完整
   - `logic`：回答是否符合逻辑
   - `readability`：回答可读性
   - `overall_score`：综合评分

---

## 实验准备

1. **设计模拟对话**：
   - 至少设计 5 个简单对话场景
   - 每个场景包含：
     - 用户输入（normal_input）
     - 模拟对话模型输出（可用真实 LLM 或 Mock）
     - 场景说明或注意事项
2. **准备输入数据**：
   - 可使用 CSV/JSON 或内置列表形式存储对话样本
3. **配置实验脚本**：
   - 参数支持选择模型、是否使用 Mock、输出目录

---

## 实验步骤

1. **加载对话样本**
   - 读取内置或外部数据
2. **模拟用户交互**
   - 对每条输入，调用 LLM 生成回答
   - 对 Mock 测试，可自定义模拟评分函数
3. **记录指标变化**
   - 对每轮对话计算 `accuracy`、`completeness`、`logic`、`readability`、`overall_score`
   - 计算 delta（如模拟对话与正常对话的差异）
4. **异常处理**
   - 记录调用失败或异常的样本
   - 允许实验继续执行其他样本

---

## 输出与分析

1. **结果落盘**：
   - CSV 文件：包含对话 ID、用户输入、模型输出、各维度评分、delta
   - XLSX 文件：同 CSV，但可直接用于分析或可视化
   - JSON 文件：结构化保存所有对话和评分结果
2. **分析报告**：
   - 计算各评分维度均值、方差
   - 统计异常对话（如 overall_score < 5）
   - 输出最大影响样本
3. **示例输出结构**：

```json
{
  "sample_id": "sim1",
  "user_input": "你好，请帮我写一首诗",
  "model_output": "春风十里不如你",
  "accuracy": 9,
  "completeness": 8,
  "logic": 9,
  "readability": 10,
  "overall_score": 9,
  "rationale": "Mock评分示例"
}
```

---

## 验收标准

- 能成功模拟至少 5 条对话
- 输出 CSV/JSON/XLSX 文件完整
- 指标计算无异常（mean/variance 合理）
- 分析报告能够标出异常对话和最大影响样本
